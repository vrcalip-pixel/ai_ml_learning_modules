üìñ Tutorial: Build a Rock-Paper-Scissors Detector ‚ñº

Goal: Train AI to recognize rock, paper, and scissors gestures.

Step 1: Go to the Website

Open teachablemachine.withgoogle.com ‚Üí Click "Get Started"

Step 2: Choose Project Type

Click "Image Project" ‚Üí "Standard image model"

Step 3: Create Classes

Rename "Class 1" to "Rock". Add two more classes: "Paper" and "Scissors"

Step 4: Collect Training Data

For each class:

  • Make the hand gesture
  • Click "Hold to Record" for 3-4 seconds
  • Move your hand slightly while recording
  • Aim for 50-100 images per class
Step 5: Train Model

Click "Train Model". Takes 30 seconds - 2 minutes.

Step 6: Test It!

Make gestures and watch the AI predict in real-time!

Step 7: Export

Click "Export Model" ‚Üí Choose "Tensorflow.js" to download

Low Accuracy?

  • Collect 100+ samples per class
  • Ensure good lighting
  • Keep background simple
  • Make gestures clearly different

Best Practices & Troubleshooting

Universal Best Practices (All Tools)

Data Quality First: Clean data before training. Remove duplicates, fix typos, handle missing values.
Start Simple: Begin with small, clear datasets. Add complexity once basics work.
Diverse Training Data: Include varied examples. Don't just use one angle, background, or scenario.
Test in Real Conditions: If your model will work in a store, test it in a store‚Äînot just your desk.
Iterate and Improve: First model won't be perfect. Collect more data for problem areas.
Document Everything: Note what data you used, settings chosen, accuracy achieved. You'll forget!

Common Problems & Solutions

üîß "My Model Has Low Accuracy" ‚ñº

Possible Causes & Fixes:

  • Not enough training data ‚Üí Collect 2-3x more samples
  • Poor data quality ‚Üí Review and clean your dataset
  • Classes too similar ‚Üí Make categories more distinct
  • Imbalanced data ‚Üí Ensure each class has similar # of examples
  • Irrelevant features ‚Üí Remove columns that don't predict outcome
üîß "Model Works in Training but Fails in Real Use" ‚ñº

This is called "overfitting":

  • Problem: Model memorized training examples instead of learning patterns
  • Fix #1: Collect more diverse training data
  • Fix #2: Test with data the model hasn't seen before
  • Fix #3: Reduce model complexity (fewer features/layers)
üîß "I Don't Have Enough Data" ‚ñº

Options:

  • Data augmentation: In Teachable Machine, rotate/flip/zoom images to create variations
  • Public datasets: Search Kaggle, Google Dataset Search for similar data
  • Start smaller: Reduce number of classes (e.g., 2 instead of 10)
  • Use pre-trained models: Hugging Face models are already trained on millions of examples
3: Notice how all these issues relate to Module 3's data quality lessons? Bad data = bad models. Always!
